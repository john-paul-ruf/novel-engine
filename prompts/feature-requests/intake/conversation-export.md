Export all agent conversations for a book as a formatted, readable document. The full record of how this book was made — the Spark brainstorming sessions, the Lumen assessments, every Forge revision plan, every Ghostlight read. Right now all of that lives in SQLite and is only accessible through the chat view one conversation at a time. After finishing a book you should be able to get that entire creative conversation out of the app.

Export as a Markdown document (or PDF via the existing Pandoc pipeline). Selectable scope: all conversations, by agent, by date range, or cherry-pick individual conversations. Each conversation gets a header with agent name, date, and pipeline phase. User and assistant messages are clearly distinguished. Thinking blocks are optionally included or suppressed. Goes wherever the user wants to save it — standard OS save dialog.

This is a zero-AI feature: pure data extraction from the existing database. Useful for process analysis (how did this revision actually unfold?), for collaborators who need to be caught up, for blogging about the experience, for archiving before the database gets pruned, and for any author who wants to study how their best revision decisions were reached. The data is already there — it just needs an exit door.
