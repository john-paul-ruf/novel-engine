Before exporting to DOCX/EPUB/PDF, authors should be able to run a pre-flight check: do all chapters exist and have content? Are there broken markdown links? Missing image references? Timeline inconsistencies? Unused scene outline entries?

A Manuscript QA feature in BuildView should scan the book and produce a report of warnings and errors that might cause rendering issues or confusion. For example: "Chapter 7 references a character not in the character profiles," or "3 images referenced in outline but not found in assets/." These aren't blockers — authors can build anyway — but they surface issues early.

This is especially critical for series, where a character introduced in Book 1 might be missing from Book 2's character profiles.
