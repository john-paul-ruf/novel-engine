The Story Bible tracks canonical plot facts, and the Motif Ledger tracks literary patterns. But there's no dedicated place to track character arcs, relationships, appearance details, and development across the manuscript. When writing a 90K novel with 15 named characters, character consistency is as critical as plot logic.

A Character Profiles section (new source file: `source/characters.json`) stores structured data for each character: appearance, voice, key relationships, emotional arc (where they start vs. end), and crucial scenes. Agents like Ghostlight and Lumen could reference this when assessing character consistency. Sable could flag continuity errors (e.g., "Eye color changed in Chapter 7").

For series authors especially, a global character registry would let you import recurring characters into new books with their existing profiles.
