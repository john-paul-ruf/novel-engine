A structured, in-story event timeline stored as `source/timeline.json` — distinct from the freeform story bible. Each entry has: event title, in-story date or time reference, chapter(s) where it occurs, characters involved, and a brief note. The UI is a simple table editor: add an event, fill in the fields, reorder by in-story time. Agents load the timeline as a source file when present.

The story bible is prose — good for character arcs, world rules, thematic statements. The timeline is structured data — good for answering "does the train journey take 4 hours or 2 hours?" and "did the protagonist's father die before or after the wedding?" Mysteries, thrillers, sci-fi, and any book with flashbacks or non-linear structure benefit enormously from a clean timeline. Even linear stories accumulate temporal inconsistencies across 80K words.

The real value is for Lumen (catching temporal contradictions during the structural assessment) and Sable (flagging timeline errors during copy edit). Both agents would read the timeline if it exists. Verity would reference it during drafting. The author maintains it — agents read but never modify it, just like the story bible. Keep the schema simple enough that a non-technical author can understand and update it.
