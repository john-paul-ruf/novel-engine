When a character's name changes or you realize a location name should be different, updating all 80 chapters one-by-one is tedious. A batch find-replace tool should work across the entire manuscript — or a subset of chapters.

This should be in the FilesView or Sidebar as a modal. Enter search term and replacement, see how many matches exist and in which chapters, then apply to all or selected chapters only. Optional: case-sensitive toggle, regex support, preview each match before committing.

Critical safety feature: every replacement should create a new chapter version (already possible via file_versions table) so authors can revert if the replace was too aggressive.
