Architecture.

130 TypeScript/TSX source files across five clean architecture layers. Dependency injection, streaming CLI integration, and strict import boundaries.

The Five Layers.

Every file belongs to exactly one layer. Import rules are strictly enforced.

LAYER 1
Domain
src/domain/ — Pure types, interfaces, constants. Zero imports. Everything depends on this.
↑ imports from
LAYER 2
Infrastructure
src/infrastructure/ — Implements domain interfaces. Isolated modules: database, filesystem, CLI client, settings, agents, providers, Pandoc.
↑ imports from
LAYER 3
Application
src/application/ — Orchestrates infrastructure through injected interfaces. 14 services.
↑ imports from
LAYER 4
Main / IPC
src/main/ + src/preload/ — Composition root (the only place concrete classes are instantiated) + thin IPC adapters.
↑ imports from
LAYER 5
Renderer
src/renderer/ — React + Zustand. 16 stores. Talks to backend only through the window.novelEngine preload bridge.

Key Design Decisions.

Constructor Injection

No DI container. Services declare interfaces in their constructor. The composition root in src/main/index.ts instantiates concrete classes and wires everything.

No API Keys Stored

The Claude Code CLI handles its own authentication through the Anthropic subscription. The app stores no secrets. OpenAI-compatible providers store API keys in user settings only.

Multi-Model Provider Architecture

A ProviderRegistry routes model requests to the correct provider. Built-in Claude CLI + any number of OpenAI-compatible endpoints. Capability-aware: providers declare what they support.

Streaming CLI Integration

Claude Code CLI outputs NDJSON events. The client parses text deltas, thinking blocks, tool use, and result events. Batched DB persistence (100ms flush, max 20 events) reduces I/O pressure.

Budget-Aware Context Assembly

Every agent call builds context using a token budget. Per-agent read guidance, file manifests with word counts, and dynamic conversation compaction (generous → moderate → tight → critical).

File Version History

Every file write is automatically snapshotted in SQLite with SHA-256 dedup. Structured diffs via the diff library. One-click revert with audit trail. Version pruning prevents unbounded storage.

Database Schema.

Seven SQLite tables. WAL mode. Foreign keys enabled. Forward-only migration system.

conversationsAll agent conversations per bookid, book_slug, agent_name, pipeline_phase, purpose, title, timestamps

Architecture Rules

These are non-negotiable for contributions:

  • Domain imports from nothing. Infrastructure imports from domain only. Application depends on interfaces, not concrete classes.
  • No business logic in IPC handlers — they are one-liner delegations to services.
  • Renderer accesses backend only through window.novelEngine. May import types from domain, never values (except pure constants/functions).
  • Constructor injection for all services. The composition root is the only place concrete classes are instantiated.
  • Every infrastructure subdirectory gets a barrel export index.ts.

Full architecture docs: docs/architecture/ on GitHub — ARCHITECTURE.md, DOMAIN.md, INFRASTRUCTURE.md, APPLICATION.md, IPC.md, RENDERER.md.
