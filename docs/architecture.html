Architecture.

Novel Engine follows Clean Architecture with five strict layers. Every file belongs to exactly one layer. Import rules are enforced by convention and code review.

The Five Layers

Domain
src/domain/
Pure types, interfaces, constants. Zero imports. Everything depends on this.
←
Infrastructure
src/infrastructure/
Concrete implementations. SQLite, filesystem, CLI wrapper, watchers.
←
Application
src/application/
Business logic. Orchestrates infrastructure through injected interfaces.
←
IPC / Main
src/main/ + src/preload/
Composition root. Thin IPC adapters. One-liner handler delegations.
←
Renderer
src/renderer/
React + Zustand. Talks to backend only through the preload bridge.

Import rule: each layer may only import from layers to its left. Domain imports nothing. Infrastructure imports domain. Application imports domain interfaces. Renderer accesses backend only through window.novelEngine.

Key Design Decisions

Constructor Injection (No DI Container)

Services receive dependencies as constructor parameters. The composition root in src/main/index.ts is the only place concrete classes are instantiated.

No API Keys

The Claude Code CLI handles authentication through the user's Anthropic subscription. The app stores zero secrets.

Streaming CLI Integration

The Claude CLI is spawned as a child process emitting NDJSON events on stdout — text deltas, thinking blocks, tool use, results. Streamed to UI in real time.

Budget-Aware Context Assembly

Every agent call builds context with a token budget. ContextBuilder assembles a file manifest, applies per-agent read guidance, and compacts conversation history.

Phase-Gated Pipeline

14 phases with file-existence completion gates. No automatic advancement — the author explicitly confirms each phase.

Verity Audit Pipeline

Automated quality pass after every agent interaction. Sonnet-powered audit scans for anti-patterns; moderate+ severity triggers automatic fixes.

Database Schema

Five SQLite tables. WAL mode. Foreign keys enforced. Forward-only migrations via migrations.ts.

Architecture Rules

  • Never put business logic in IPC handlers — they are one-liner delegations
  • Never import concrete classes in application services — use injected interfaces
  • Never let the renderer import from main process modules — only through the preload bridge
  • Never store API keys — the Claude Code CLI handles auth
  • Always use parameterized SQL queries — never interpolate values
  • Every infrastructure subdirectory gets a barrel index.ts
