Press Kit.

For journalists, reviewers, newsletter writers, and anyone writing about the project.

The Pitch

Novel Engine is to book-writing what a build system is to software. It’s not a chatbot. It’s not an AI ghostwriter. It’s an editorial production pipeline where the human author retains creative authority and seven specialized AI agents serve as the author’s professional editorial team.

The pipeline takes a book from pitch to polished manuscript in 14 structured phases. Each phase has a designated agent, clear inputs and outputs, and a completion gate the author explicitly confirms. The result: five published novels on Amazon, independently scored 7.0–9.4 out of 10 on an “AI slop vs. established author” scale by both Claude and ChatGPT.

What Makes This Different

Build system, not chatbot

14-phase pipeline with completion gates, not freeform prompting. Each phase has defined inputs, outputs, and a designated specialist agent.

7 specialized agents

Not one general AI, but a team with distinct roles: ideator, ghostwriter, first reader, developmental editor, copy editor, task master, query crafter.

Voice capture

Writes in the author’s voice via a detailed profile interview. Sentence rhythm, vocabulary register, dialogue style, emotional temperature — all captured before a single word of prose.

Local-first

Everything runs on your machine. No cloud backend. No API key storage. The Claude Code CLI handles authentication through the user’s own subscription.

Open source

AGPL-3.0 licensed. Fully inspectable. 121 TypeScript source files across five clean architecture layers.

Ships real books

5+ published novels on Amazon built with this system. Not a prototype — a production tool with published proof.

Quality automation

Every chapter runs through an automated audit pipeline. A Sonnet-powered agent scans for anti-patterns; moderate+ issues trigger automatic fixes.
