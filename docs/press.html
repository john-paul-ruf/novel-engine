Press Kit.

For journalists, reviewers, newsletter writers, and anyone writing about Novel Engine.

The Pitch.

Novel Engine is to book-writing what a build system is to software. It's a desktop application that organizes the entire editorial lifecycle — from initial concept through polished manuscript — into a structured, phase-gated pipeline powered by seven specialized AI agents.

It is not a chatbot. It is not an AI ghostwriter. It's an editorial production pipeline where the human author retains full creative authority. Each agent has a distinct role — ideation, ghostwriting, cold reading, developmental editing, copy editing, task planning, and publishing — and operates at precisely the right moment in the production process. The author drives every decision; the agents are the editorial staff.

What Makes This Different.

Architecture

Build system, not chatbot

14-phase pipeline with explicit completion gates. Each phase has defined inputs, outputs, and a designated agent. The author confirms advancement — no automatic progression. This is structured production, not freeform prompting.

Design

7 specialized agents

Not one general AI doing everything. Spark ideates. Verity drafts. Ghostlight cold-reads. Lumen runs structural analysis. Sable copy-edits. Forge plans revisions. Quill prepares publication metadata. Each agent has distinct instructions, thinking budgets, and context rules.

Innovation

Voice capture

Before drafting, an interview process captures the author's writing voice — cadence, vocabulary, sentence structures, stylistic preferences. Verity then drafts prose in that captured voice, not generic AI-speak.

Privacy

Local-first

Everything runs on your machine. No cloud backend. No API key storage. The Claude Code CLI handles its own authentication. Your manuscripts, notes, and conversations never leave your computer.

Transparency

Open source

AGPL-3.0 licensed. 158 TypeScript source files. ~44,000 lines of code. Full architecture documentation. Every design decision is inspectable, from the 5-layer clean architecture to the streaming CLI integration.

Proof

Ships real books

Five published novels on Amazon, all built with this system. Independently evaluated by two AI reviewers (Claude and GPT) with scores ranging 7.0–9.4 out of 10. Not AI slop — actual books people read.

Flexibility

Multi-model support

Pluggable provider architecture supports Claude Code CLI natively plus any OpenAI-compatible API endpoint. Capability-aware model routing, per-agent model assignment, and provider health monitoring.
