Contributing.

Novel Engine welcomes contributions. The codebase follows strict clean architecture — understanding the layer boundaries is the most important thing before writing code.

Getting Started

  1. Fork the repo and clone it locally
  2. Install prerequisites: Node.js 20+, Claude Code CLI (installed and authenticated)
  3. Run npm install then npm run download-pandoc
  4. Start in dev mode with npm start
  5. Create a branch, make your changes, open a PR

Architecture Rules

These are non-negotiable. PRs that violate them will be asked to refactor:

  • Domain imports from nothing. Pure types, interfaces, and constants only. Zero imports.
  • Infrastructure imports from domain only. Each module is isolated — database doesn't import from filesystem, etc.
  • Application depends on interfaces, not concrete classes. Constructor injection. No new ConcreteClass() in service code.
  • No business logic in IPC handlers. Every handler is a one-liner delegation to a service method.
  • Renderer accesses backend only through window.novelEngine. May import types from domain, never values (except pure constants).
  • Every infrastructure subdirectory gets a barrel export. index.ts that re-exports the public API.

Read the full architecture docs: Architecture page or the Markdown docs on GitHub.

Report a Bug.

Open a new issue on GitHub with the following:

Platform: macOS / Windows / Linux (version)
App Version: v0.1.0 (or commit hash)
Steps to reproduce: What you did
Expected: What should have happened
Actual: What actually happened
Console output: Any error messages (Cmd+Opt+I / Ctrl+Shift+I to open DevTools)

Testers Wanted.

Pre-built installers need testing!

Installers are available on the Releases page. The developer builds on macOS — Windows and Linux installers especially need eyes on them.

macOS .dmg Windows Squirrel Linux .deb

Things to test:

  • Does the installer run and complete without errors?
  • Does the app launch after installation?
  • Does the onboarding wizard detect your Claude Code CLI?
  • Can you create a book and chat with an agent?
  • Any UI glitches, missing fonts, or broken layouts?

Report results via GitHub Issues or email john.paul.ruf@gmail.com.
