Contributing

Novel Engine welcomes contributions. The codebase follows strict clean architecture — understanding the layer boundaries is essential before writing code.

Start Here

  • Read the Architecture page to understand the 5-layer structure
  • Fork the repo and clone it locally
  • Install prerequisites: Node.js 20+, Claude Code CLI (installed and authenticated)
  • Run npm install && npm run download-pandoc && npm start
  • Find work on the Issues page
  • Create a branch, make your changes, and submit a PR

Architecture Rules

  • Respect layer boundaries — Domain imports nothing. Infrastructure imports domain only. Application depends on interfaces, not concrete classes. Renderer talks to backend only through the preload bridge.
  • No business logic in IPC handlers — Handlers are one-liner delegations to services.
  • Dependency injection — Services are composed in src/main/index.ts. Never instantiate services elsewhere.
  • Parameterized SQL — Never interpolate values into SQL strings.
  • Barrel exports — Every infrastructure subdirectory gets an index.ts.
  • No API keys — The Claude Code CLI handles authentication. The app stores no secrets.

Report a Bug

Open an issue on GitHub: New Issue

Please include:

  • Platform — macOS, Windows, or Linux (with version)
  • App version — from Settings or the title bar
  • Steps to reproduce — what you did, what you expected, what happened
  • Console output — if available (DevTools → Console tab)
