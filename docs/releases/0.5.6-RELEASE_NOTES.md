# Release Notes ‚Äî v0.6.0

**Previous release:** v0.5.4 (2026-03-25)
**This release:** v0.6.0 (2026-03-27)
**Commits:** 42 | **Files changed:** 172 | **Contributors:** the.phoenix

---

## Highlights

> Novel Engine's largest architectural release to date. The monolithic ChatService (1,218 lines) has been decomposed into 6 focused services ‚Äî StreamManager, AuditService, PitchRoomService, HotTakeService, and AdhocRevisionService ‚Äî with proper domain interfaces for each. A new **Motif Ledger** view gives writers a structured, tab-based editor for tracking motif systems, foreshadow threads, character entries, and flagged phrases across their entire manuscript. The Verity agent has been deconstructed from a single prompt into 9 purpose-built sub-prompts (draft, revision, audit, scaffold, etc.), and a formal database migration system replaces ad hoc schema patches. Three rounds of repo evaluation and bug fixes resolved race conditions in concurrent streaming, error-path cleanup, and DOM nesting warnings.

---

## Features

- **Motif Ledger view** ‚Äî Full-stack feature: domain types, application service, IPC wiring, Zustand store, and 7 CRUD tab panels (Systems, Entries, Structural, Foreshadow, Minor Characters, Flagged Phrases, Audit Log). Cmd+S save shortcut, dirty tracking, unaudited chapter warnings. (#7d65e5c, #0b91c79, #4bb689b)
- **Pitch Room history** ‚Äî Pitch Room conversations now persist and are browsable via a new `PitchHistory` sidebar component. (#495da3b, #793351a)
- **Auto-draft UI improvements** ‚Äî Auto-draft stages two and three show real-time progress in conversations. Phase completion now waits for all phases before proceeding. (#3e73451, #ced6ba0, #e45e68f)
- **GitHub Pages website** ‚Äî 6-page site: landing page, 10-book evaluation, architecture docs, changelog, press kit, and contact page. Dark theme with agent color coding, responsive design, no external JS. (#a7a7c4b)

## Improvements

- **Verity agent pipeline** ‚Äî Deconstructed monolithic VERITY.md into 9 purpose-built sub-prompts: VERITY-CORE, VERITY-DRAFT, VERITY-REVISION, VERITY-AUDIT, VERITY-FIX, VERITY-SCAFFOLD, VERITY-MECHANICAL, VERITY-LEDGER, and VERITY-LEGACY. Each prompt is scoped to a specific pipeline phase. (#c83e3ac, #8cfaa11, #471a0ec)
- **Forge agent rewrite** ‚Äî Expanded FORGE.md with improved developmental assessment instructions. (#e1bed3b)
- **Phrase-to-motif consolidation** ‚Äî Eliminated standalone `phrase-ledger.md` artifact. All phrase/repetition tracking now lives in `motif-ledger.json`. Renamed `phrase-ledger-hit` ‚Üí `flagged-phrase` across types, agents, and UI. (#47dc418)
- **Prose quality** ‚Äî Upstream fixes to reduce repeated linguistic constructs before they require revision. (#5f714c1, #3f79144)
- **CLI kill-on-demand** ‚Äî Users can now forcefully terminate an active Claude CLI process. (#a2f2bc8)
- **Bash and delete tool permissions** ‚Äî Added `bash` and `delete` tool permissions to CLI invocation for agents that need them. (#32abe05, #ea85610)
- **Promote pitch to book** ‚Äî Pitch Room conversations can now be promoted into a new book project. (#6109f33)
- **README rewrite** ‚Äî Comprehensive update: corrected agent budgets, added all new services, updated source tree to 121 files. (#253e151)

## Bug Fixes

- **Startup crash: MotifLedgerView** ‚Äî Tab count computation crashed on `undefined.length` when ledger JSON had missing arrays. Added optional chaining guards. (#bb5711a)
- **Nested button DOM warning** ‚Äî BookSelector's dropdown trigger changed from `<button>` to `<div role="button">` to fix React `validateDOMNesting` error. (#bb5711a)
- **SystemsTab crash** ‚Äî `sys.components` undefined from partially-populated ledger JSON. Added `?? []` fallbacks. (#bb5711a)
- **Pitch Room bug** ‚Äî Fixed issue with pitch room message handling. (#a9153b6, #aa13e2f)
- **Sandbox issues** ‚Äî Resolved CLI sandbox permission errors. (#f11c58f)
- **Thinking budget misconfiguration** ‚Äî Fixed incorrect thinking budget values being sent to agents. (#236e3f9)
- **Race conditions in concurrent streams** ‚Äî Book switching now only kills `'self'` streams, preserving background auto-draft/hot-take streams. Added `_streamOrigin` discriminator. (#e4c7d32)
- **Stale `_activeCallId` on error** ‚Äî Error catch blocks now clear `_activeCallId` and filter temp messages across chatStore, modalChatStore, pitchRoomStore. (#44cb145)
- **Revision event forwarding** ‚Äî Added missing `conversationId` to `session:streamEvent` variant. (#44cb145)
- **Missing `callStart` events** ‚Äî Verity audit/fix calls now emit synthetic `callStart` events so the CLI activity monitor tracks them. (#44cb145)
- **Duplicate polling intervals** ‚Äî cliActivityStore recovery uses module-level timer refs to prevent stacking. (#44cb145)
- **Silent EPIPE swallowing** ‚Äî ClaudeCodeClient now logs `stdinBytes`, `writableFinished`, `writableEnded` on EPIPE. DB persistence errors logged on first failure per session. 500KB system prompt size guard. (#44cb145)

## Refactoring

- **ChatService decomposition** ‚Äî Extracted 5 services from the 1,218-line ChatService monolith, reducing it to a 403-line router (67% reduction):
  - `StreamManager` (219 lines) ‚Äî Active stream lifecycle, buffer accumulation, cleanup (#193fec0)
  - `AuditService` (350 lines) ‚Äî `auditChapter()`, `fixChapter()`, `runMotifAudit()` (#7d65e5c)
  - `PitchRoomService` (109 lines) ‚Äî Pitch room message handling (#7d65e5c)
  - `HotTakeService` (98 lines) ‚Äî Hot take assessment flow (#7d65e5c)
  - `AdhocRevisionService` (105 lines) ‚Äî Direct feedback revision flow (#7d65e5c)
- **Domain interfaces** ‚Äî Added `IChatService`, `IUsageService`, `IAuditService`, `IPitchRoomService`, `IHotTakeService`, `IAdhocRevisionService`. IPC handlers now depend on abstractions, not concrete classes. (#7d65e5c)
- **`resolveThinkingBudget()` extraction** ‚Äî Pure function moved to `src/application/thinkingBudget.ts`. (#193fec0)
- **Status messages extraction** ‚Äî ~190 lines of status message arrays moved from `constants.ts` to new `statusMessages.ts`. constants.ts reduced from 755 to 273 lines. (#7d65e5c)
- **Prompt template extraction** ‚Äî 9 prompt template strings moved from `constants.ts` to standalone `.md` files in `agents/`. Loaded at runtime via `AgentService.loadRaw()`. (#7d65e5c)
- **Shared stream handler** ‚Äî `createStreamHandler()` factory deduplicates event dispatch logic across chatStore, modalChatStore, pitchRoomStore. Replaced `streamRouter.ts`. (#926efeb, #44cb145)
- **Stream event batching** ‚Äî ClaudeCodeClient batches DB persistence (100ms flush, max 20 events, critical events flush immediately). New `persistStreamEventBatch()` on `IDatabaseService`. (#e4c7d32)
- **Wrangler role update** ‚Äî Renamed from 'Context Planner' to 'Revision Plan Parser' to reflect actual usage. (#7d65e5c)
- **Silent error audit** ‚Äî Added explanatory comments to 12 uncommented catch blocks across 6 files. (#7d65e5c)

## Breaking Changes

- **`IChatService.sendMessage` return type** ‚Äî Changed from `Promise<void>` to `Promise<{ changedFiles: string[] }>`. Callers must handle or ignore the new return value.
- **`IChatService.getLastChangedFiles()` removed** ‚Äî Use the return value from `sendMessage()` instead.
- **IPC channel rename** ‚Äî `verity:runPhraseAudit` ‚Üí `verity:runMotifAudit`
- **Bridge method rename** ‚Äî `window.novelEngine.verity.runPhraseAudit` ‚Üí `runMotifAudit`
- **Domain type rename** ‚Äî `AuditViolationType` variant `'phrase-ledger-hit'` ‚Üí `'flagged-phrase'`
- **New database table** ‚Äî `schema_version` (version INTEGER, applied_at TEXT, description TEXT) created automatically on startup.
- **`streamRouter.ts` deleted** ‚Äî Replaced by `streamHandler.ts` with `createStreamHandler()` factory.

## Upgrade Notes

1. **Database auto-migrates.** The new `schema_version` table is created automatically. No manual SQL needed.
2. **Agent files auto-rename.** `FORGE.MD` ‚Üí `FORGE.md` and `Quill.md` ‚Üí `QUILL.md` happen in bootstrap. No action needed.
3. **New agent files auto-copy.** The 9 new prompt files (VOICE-SETUP.md, PITCH-ROOM.md, etc.) and 9 Verity sub-prompts are copied to `custom-agents/` on startup via `COPYFILE_EXCL` (won't overwrite existing customizations).
4. **Phrase ledger deprecated.** Existing `source/phrase-ledger.md` files are ignored. Migrate data to `motif-ledger.json` `flaggedPhrases` section manually if desired.
5. Pull and rebuild. No other manual steps required.
