## Highlights

> You can now **import an existing manuscript** — drop in a `.md`, `.txt`, or `.docx` file and Novel Engine will detect chapter boundaries, let you review and edit them, then create a full book project in one step. After import, optional AI-powered source document generation (pitch, outline, bible, voice profile, motif ledger) gets your imported book up to speed with books started natively. The Motif Ledger also got a major reliability fix: malformed agent-written JSON is now normalized via a Sonnet CLI call instead of brittle regex, eliminating data corruption on load.

---

## Features

- **Manuscript import wizard** — Import `.md`, `.markdown`, `.txt`, or `.docx` files into Novel Engine. Automatic chapter detection via heading patterns and "Chapter N" matching with ambiguity warnings for uneven splits. Preview UI with inline rename, merge, and remove controls. Creates full book directory structure with status set to `first-draft`. (#82c0d51, #2c76830, #93f9b1f)

- **Post-import source generation** — After importing a manuscript, optionally trigger 4 sequential agent calls (Spark for pitch, Verity for outline+bible, voice profile, and motif ledger) with per-step progress tracking. Brings imported books to feature parity with natively-created ones. (#82c0d51, #2c76830, #93f9b1f)

- **Feature scouting framework** — New `prompts/meta/feature-scout.md` meta-prompt for systematically evaluating feature ideas. 8 feature request stubs generated: agent prompt editor, chapter reorder, conversation search, keyboard shortcuts, manuscript import, series bible, writing goals, and writing statistics dashboard. (#aec7819, #0c5a27b)

## Bug Fixes

- **Motif Ledger JSON corruption** — Replaced brittle hardcoded field-mapping normalizers with a CLI-based normalization step. Non-canonical JSON shapes (agent-written fields like `associatedCharacters`, object-typed `firstAppearance`, `plant`/`payoff` foreshadow objects) are now sent to a Sonnet CLI call with the full target schema. Normalized result is saved back to disk so normalization only fires once. Falls back to best-effort parse if the CLI call fails. UI shows a spinner during normalization. (#50588e2)

- **Press page rendering** — Fixed broken layout in the GitHub Pages press kit page. (#03eb1e7)
