## Highlights

> Novel Engine gains **series support**, **batch manuscript import**, an **in-app help assistant**, and a **guided tour & tooltip system**. You can now group books into ordered series with a shared story bible that's automatically injected into agent context. Importing multiple manuscripts at once creates a series in one shot. A floating helper agent answers questions about the app using a comprehensive user guide. And new users are welcomed with interactive guided tours and contextual tooltips across every major UI element.

---

## Features

- **Series Bible** — Group books into ordered series with shared story bibles. File-based storage (`{userData}/series/{slug}/`), reverse-lookup cache for O(1) book→series resolution, automatic series bible injection into all 7 creative agent contexts. Full CRUD with SeriesModal, VolumeList, and SeriesBibleEditor components. 11 new IPC channels. (#c5f8c83)

- **Series Import** — Batch manuscript import with series grouping. Select multiple files, preview as ordered volumes, edit titles, reorder, skip individual volumes, and commit to a new or existing series. Composes existing ManuscriptImportService and SeriesService through a new SeriesImportService. (#fdd986a, #17a99dd)

- **In-App Helper Agent** — Non-creative Helper agent accessible via floating chat bubble (bottom-right corner). Uses a comprehensive 16-section user guide as its knowledge base. Answers questions about features, workflows, agents, and troubleshooting. Conversation persists across book switches and view changes. New `HelperService`, `helperStore`, and 5 `helper:*` IPC channels. (#0877e39, #45e680a)

- **Guided Tours** — Three interactive spotlight-based tours (Welcome, First Book, Pipeline Intro) with CSS clip-path cutouts, keyboard navigation, and step-by-step popovers. Welcome tour auto-launches after onboarding. Replay from Settings. GuidedTourOverlay handles window/element resize. (#c9418e9, #831f894, #f034f80, #7929776)

- **Contextual Tooltips** — Portal-based tooltip component with configurable placement, delay, and arrow. Applied to 14 components: all sidebar buttons, pipeline phases, nav items, chat controls, file view toggles, and window controls. Suppressed during active tours. Accessible via `aria-describedby`. (#c9418e9, #5c658ee)

- **Sidebar Help Button** — "?" button in sidebar header with popover to launch guided tours on demand. (#7929776)

## Improvements

- **Series-Aware Book Selector** — BookSelector now groups books by series with collapsible SeriesGroup headers. Standalone books shown separately. "Manage Series" and "Import Series" buttons added. (#c5f8c83, #fdd986a)

- **Agent Context: Series Bible Injection** — ContextBuilder accepts `seriesBiblePath` param; ChatService resolves series bible before context assembly. All 7 creative agents have `series-bible.md` in their `readIfRelevant` guidance. (#c5f8c83)

- **Helper Agent System Prompt** — New `agents/HELPER.md` with behavior rules and conversation style. `HELPER` entry in AGENT_REGISTRY (blue-500 color, 2K thinking budget, 5 max turns). (#0877e39)

- **User Guide** — Comprehensive `docs/USER_GUIDE.md` covering all 16 feature areas. Copied to userData on every startup via `ensureUserGuide()` in bootstrap. Bundled in packaged builds via `extraResource`. (#0877e39)

- **Tour Completion Persistence** — `AppSettings` extended with `completedTours: TourId[]`. Tour state persisted through existing `settings:update` IPC channel. (#c9418e9)

## Upgrade Notes

No special upgrade steps required. All new features are additive:

1. **Series directory** — Created automatically by bootstrap on first run after upgrade.
2. **`completedTours`** — Defaults to `[]` in settings; existing settings files backfilled by merge logic.
3. **Helper agent** — `HELPER.md` and `USER_GUIDE.md` deployed automatically on startup.
4. **No database schema changes** — Series data is file-based, tour data uses existing settings.

Pull and rebuild. No migrations needed.
