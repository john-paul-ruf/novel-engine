## Highlights

> Novel Engine now supports **multiple AI providers** — bring your own API key for any OpenAI-compatible endpoint (GPT-4, Gemini, Mistral, local models) alongside the built-in Claude CLI. A full **content version control** system tracks every file change with diffs and one-click revert, so you never lose a draft revision again. You can also **export your entire book catalog** as a single ZIP archive from Settings. Several data-corruption bugs in the Motif Ledger were fixed, including one that caused total data loss of the ledger file.

## Features

- **Multi-model provider architecture** — Pluggable AI provider system. Register any OpenAI Chat Completions-compatible endpoint alongside Claude CLI. Provider management UI in Settings with status indicators, connectivity testing, and model grouping by provider. Models without tool-use support are badged as "Text only". 7 sessions: domain types → ClaudeCodeClient adapter → ProviderRegistry → OpenAiCompatibleProvider → service migration → IPC/preload → renderer UI. (#8ddd25f, #0e3d70b, #5057505, #7156547, #8e65ece, #67cbe3d, #99861b3)

- **Content version control** — Automatic snapshot-per-write versioning for all `.md` and `.json` files. SHA-256 dedup prevents duplicate snapshots. Structured diffs with unified diff viewer (color-coded additions/deletions, hunk headers, line numbers). Version history timeline in a slide-over panel accessible from FileEditor, FilesView reader mode, SourcePanel, ChaptersPanel, and AgentOutputPanel. One-click revert with confirmation. Auto-snapshots at 5 capture points: user edits, pipeline agent writes, hot takes, adhoc revisions, and revision queue execution. Startup pruning keeps storage bounded. 7 sessions: domain types → DB migration → VersionService → IPC wiring → store + DiffViewer → VersionHistoryPanel → integration. (#e84a5c6, #73e062c, #ed9b0f0, #43517ba, #f24e0f5, #7309593, #752e65e)

- **Catalog export** — Export the entire `books/` directory as a timestamped ZIP archive from Settings. Uses the existing `archiver` dependency. (#89ba0f0)

- **Intake meta-prompt** — New `prompts/meta/intake.md` for decomposing feature specs, research docs, and raw ideas into ordered session prompts with MASTER/STATE/SESSION file generation. (#906a704)

## Improvements

- **ProviderSection in Settings** — New dedicated section for managing providers between CLI status and model selection. Cards with green/red/gray status dots, add/remove/toggle, and test connectivity buttons. (#99861b3)

- **Model selection grouped by provider** — Models in Settings now grouped under their provider name with "Text only" badges for non-tool-use models. Selecting a model from a different provider auto-updates `activeProviderId`. (#99861b3)

## Bug Fixes

- **MotifLedgerService data loss** — The JSON repair regex matched `}{` patterns inside string values (not just between array elements), corrupting parsed structure. Auto-writeback then overwrote the 133KB original with an empty/corrupt version. Fixed by removing auto-writeback entirely (load is now read-only) and rewriting `repairJson()` to line-by-line structural repair. (#4ffc63f, #906a704)

- **Audit Log tab crash** — The MOTIF-AUDIT agent writes records with `{ chapter, date, findings }` but the UI expected `{ id, chapterSlug, auditedAt, ... }`. Sort called `.localeCompare()` on `undefined`. Fixed by normalizing agent-written data on load and adding defensive fallback in the sort. (#edffe7a)

- **Hot Take button not appearing after chapter creation** — Button only re-checked for chapters when `activeSlug` changed, not when files were created on disk. Fixed by subscribing to `fileChangeStore.revision`. (#60a252e)

## Breaking Changes

- **`IClaudeClient` deprecated** — All application services now depend on `IProviderRegistry` instead. `IClaudeClient` is retained for backward compatibility but marked deprecated. Code referencing `IClaudeClient` directly should migrate to `IProviderRegistry`.

- **`AVAILABLE_MODELS` deprecated** — Replaced by `providerRegistry.listAllModels()` which returns `ModelInfo[]` (a superset of the old shape). `settings:getAvailableModels` IPC channel now returns `ModelInfo[]`.

- **`AppSettings` extended** — New fields `providers: ProviderConfig[]` and `activeProviderId: ProviderId`. Existing settings files get defaults from `DEFAULT_SETTINGS` merge automatically.

- **Database schema v2** — New `file_versions` table with composite indexes. Migration runs automatically on startup. Non-destructive — no changes to existing tables.

---

## Upgrade Notes

1. Pull latest and run `npm install` — new `diff` package dependency.
2. Database migration v2 runs automatically on first launch — creates `file_versions` table and indexes. No data loss.
3. Existing `settings.json` files missing `providers`/`activeProviderId` will get defaults automatically on load.
4. If you were importing `IClaudeClient` in custom code, migrate to `IProviderRegistry`. The old interface still works but is deprecated.
5. Renderer code using `models.getAvailable()` now receives `ModelInfo[]` instead of `{id, label, description}[]`. The fields are a superset, so existing destructuring continues to work.
